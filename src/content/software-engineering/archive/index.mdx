# Archive

## Summary

- Preserve old notes here instead of deleting them.
- Archived notes should keep original meaning and important context.
- Sensitive archived notes should remain private.

## Mental Model

The archive keeps older material available without mixing it into active learning paths or project dashboards.

## Details

Move notes here when they are outdated, superseded, or no longer useful in the main navigation.

## Example

- Old implementation notes.
- Superseded project plans.
- Deprecated study material.

## Common Mistakes

- Deleting content that still has historical context.
- Leaving archived notes marked as published when they are no longer reliable.
