# Drafts

## Summary

- Put incomplete, messy, sensitive, or unclear notes here first.
- Work-related drafts should stay private until reviewed.
- Redact secrets and private implementation details before moving notes into public areas.

## Mental Model

Drafts are the intake queue for the knowledge base.

## Details

Use this folder when the category is unclear, the content needs cleanup, or the note contains sensitive details.

## Example

```txt
apiKey: "<redacted-api-key>"
customerId: "<redacted-customer-id>"
```

## Common Mistakes

- Publishing work details before sanitizing them.
- Leaving useful notes outside the content database.
- Creating notes without complete frontmatter.
