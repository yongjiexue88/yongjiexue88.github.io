# Learning

## Summary

- Use this area for general technical concepts and study notes.
- Prefer clear mental models, production considerations, tradeoffs, and examples.
- Put messy or sensitive material in [[Drafts]] first.

## Simple Explanation

Learning notes explain concepts in a way that can be reviewed quickly and connected into a roadmap.

## How It Works Internally

Each note should include complete frontmatter and enough structure for the frontend to create category pages, tag filters, and related-note sections.

## Production Considerations

- Keep examples realistic.
- Mark uncertain notes as drafts.
- Avoid exposing sensitive work details in public learning notes.

## Tradeoffs

- Short notes are easier to review.
- Deeper notes are better for interview preparation and future implementation work.

## Interview Framing

Learning notes should include interview framing when the topic commonly appears in interviews.
